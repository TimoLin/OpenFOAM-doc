Detailed Description

This boundary condition specifies a velocity inlet profile appropriate for atmospheric boundary layers (ABL). The profile is derived from the friction velocity, flow direction and the direction of the parabolic co-ordinate z.

where

= frictional velocity
= Karman's constant
= vertical co-ordinate [m]
= surface roughness length [m]
= minimum vlaue in z direction [m]

where:

= reference velocity at [m/s]
= reference height [m]

Reference: D.M. Hargreaves and N.G. Wright, "On the use of the k-epsilon model in commercial CFD software to model the neutral atmospheric boundary layer", Journal of Wind Engineering and Industrial Aerodynamics 95(2007), pp 355-369.


Patch usage

Property Description Required Default value
n flow direction yes
z vertical co-ordinate [m] yes
kappa Karman's constanat no 0.41
Uref reference velocity [m/s] yes
Href reference height [m] yes
z0 surface roughness length [m] yes
zGround minimum z co-ordinate [m] yes

Example of the boundary condition specification: 

    myPatch
    {
        type            atmBoundaryLayerInletVelocity;
        n               (0 1 0);
        z               1.0;
        kappa           0.41;
        Uref            1.0;
        Href            0.0;
        z0              uniform 0.0;
        zGround         uniform 0.0;
    }
Note
D.M. Hargreaves and N.G. Wright recommend Gamma epsilon in the k-epsilon model should be changed from 1.3 to 1.11 for consistency. The roughness height (Er) is given by Er = 20 z0 following the same reference.
